# BRIEF — CraftPG (Tormenta20 Campaign Manager)

## Problem (1 sentence)
Tormenta20 groups manage campaigns, sessions, and character sheets across generic tools (Docs/Notion/Discord), lacking role-based collaboration, invite flow, and a real visual character sheet with per-campaign progression.

## Solution
A web app to create Tormenta20 campaigns and sessions, invite users by email with roles (DM/Moderator/Player), and require each participant to join with a character. The system provides a visual “real sheet-like” character sheet that evolves over time, supports rule-guided level-up, and keeps progression independent per campaign.

## Target users
- Primary: Dungeon Masters (DMs) running Tormenta20 campaigns
- Secondary: Players maintaining their character progression and session records

## Differentiators
- Participation is character-based: no user exists in a campaign without a character
- Character is locked per campaign (no switching after joining)
- Same character concept can exist across different campaigns with different levels/progression
- Session-level content: NPCs, maps, treasures, XP, attendance

## Business model (optional)
Freemium:
- Free: limited campaigns/sessions/storage
- Pro: unlimited campaigns, PDF export, advanced sheet automation, shared libraries, templates

## Success metrics
- Activation: % of new users who create a campaign + create a first session in 24h
- Invite conversion: invites sent → accepted
- Time-to-join: invite received → member joined with character
- Retention: DMs returning weekly (4-week retention)
- Engagement: campaigns with ≥ 3 sessions created
